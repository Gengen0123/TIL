## 今日触れたこと
- git diffの使い方
  - ファイルに変更を加えてからgit diffを実行すると、変更箇所がわかる。
  - 変更を加えたファイルをgit add 変更したファイル名　でステージに上げる
  - ここでgit diff --stagedを使うと、ステージにあるファイルの変更箇所がわかる

- git logの使い方
  - git logで変更履歴を直近のものから順に確認できる
  - git log --onelineで各変更履歴を一行で表示できる
  - git log -p 変更したファイル名で差分のみ表示できる
  - git lon -n＜コミット数＞で最近の変更だけ確認できる

- git rmの使い方
  - ファイルごと消したいとき
    - git rm ファイル名
    - git rm -r　ディレクトリ名
  - ファイルを残したいとき
    - git rm --cached　ファイル名
